A kernel page table per process

Xv6有一个单独的内核页表，每当它在内核中执行时都会使用它。内核页表是到物理地址的直接映射，因此内核虚拟地址 x 映射到物理地址也是 x 。Xv6还为每个进程的用户地址空间提供单独的页表，仅包含从虚拟地址0开始的该进程的用户内存的映射。因为内核页表不包含这些映射，所以用户地址在内核中是not valid 的。因此，当内核需要使用在系统调用中传递的用户指针（例如，传递给write（）的缓冲区指针）时，内核必须首先将指针转换为物理地址。本节和下一节的目标是允许内核直接对用户指针进行解引用。

现在的任务是修改内核让每个进程在内核中执行时用它自己对内核页表的copy，而不需要主内核页表进行地址转换

实际上，存在有两种方案：

1、为每个进程copy一个内核页表

2、共享内核页表，share all the entries that are basically are going to be unmodfied

在这里，我们用的是第一种方案
